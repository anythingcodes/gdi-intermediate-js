Review

Arrays

Syntax

  • Wrapped in square brackets
    • var courses = [];
  • Each value is separated by a comma:
    • var courses = ['JS', 'WordPress', 'ES6'];
  • Access values by zero-based index using square bracket notation:
    • var courses = ['JS', 'WordPress', 'ES6']; console.log(courses[0]); // JS console.log(courses[1]); // WordPress console.log(courses[2]); // ES6

Adding & Removing Values

  • Add a value by calling push():
    • var courses = ['JS', 'WordPress', 'ES6']; courses.push('Sketch'); console.log(courses); // ['JS', 'WordPress', 'ES6', 'Sketch']
  • The pop() method removes the last item from the array and returns it:
    • var courses = ['JS', 'WordPress', 'ES6', 'Sketch']; var removedCourse = courses.pop(); console.log(courses); // ['JS', 'WordPress', 'ES6'] console.log(removedCourse); // 'Sketch'
Point out console.log()! We'll be exploring this in detail as part of next section but, for now, note that it prints out the value so we as developers can see the info but it isn't shown on the webpage itself.

Object Syntax

  • Wrapped in curly braces
    • var gdiCourse = {};
  • Each property is a key and value pair, separated by a colon:
    • var gdiCourse = { name: 'Intermediate JavaScript' };
  • Any additional property is separated by a comma:
    • var gdiCourse = { name: 'Intermediate JavaScript', level: 2 };

Review

Accessing Object Values

var gdiCourse = {
  name: 'Intermediate JavaScript',
  level: 2
};
  • Access values by key using square bracket or dot notation:
    • var gdiCourse = { name: 'Intermediate JavaScript', level: 2 }; console.log(gdiCourse.name); // Intermediate JavaScript console.log(gdiCourse['level']); // 2

    Note the quotes around the key name in bracket notation

  • Functions can be added to objects, but are called like standard functions:
    • var gdiCourse = { name: 'Intermediate JavaScript', level: 2, logCourseName: function() { console.log(this.name + ' is level ' + this.level); } }; gdiCourse.logCourseName(); // Intermediate JavaScript is level 2
Point out console.log()! We'll be exploring this in detail as part of next section but, for now, note that it prints out the value so we as developers can see the info but it isn't shown on the webpage itself.

Breakpoints

Console statements are helpful, but what if we want to track code, step by step?

Breakpoints pause ⏸️ code execution, allowing you to capture what's going on at that moment in time

How to set a breakpoint

Let's take a look »

  1. Find your JS file in the Sources tab of dev tools
  2. Click the line number to the left of any lines to which you'd like to add breakpoints
  3. Reload the page
  4. Click the Continue icon () to continue from one breakpoint to the next, or the Step Into icon () for a more granular step-by-step view

You can also set conditional breakpoints by right-clicking a breakpoint and clicking Edit breakpoint...

Show how totalFunctionsCalled changes. Set a conditional breakpoint too tied to myNumber value. Show how 'Step Into' steps into the function, which is helpful to get a line-by-line recording of what's happening.
  • Step Over - step through code line by line to get insights into how each line affects the cars being updated. If your code calls anoter func, the debugger won't jump into its code, instead stepping over so that the focus remains on the current function/scope.
  • Step Into - like step over, however clicking step into at the function call will cause debugger to move its execution to the first line in the functions definition
  • Step Out - having stepped into a function, clicking this will cause the remainder of the function definiton to be run and the debugger will move its execution to the parent function

Events & DOM Interactions

Events

When someone interacts with your page, how do you control what happens?

You'll need 4 things:

  1. The DOM element you're monitoring
  2. A string of the type of event you're monitoring on this DOM element, such as:
    • When this link element is 'click'ed
    • When this input is 'change'd
    • When this window is 'scroll'ed
  3. A callback function, which happens once this type of event fires off. The callback function contains the code you'd like to run once this event happens.
  4. The addEventListener() method to bring it all together

Syntax

Let's say we have a checkbox element in our HTML:

 

We would add an event listener to our JS like so:

myCheckbox.addEventListener('change', function() {
  console.log('The myCheckbox element changed!');
});
